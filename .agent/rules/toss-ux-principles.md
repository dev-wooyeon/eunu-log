# Toss UX Principles & Interaction Patterns

## 1. UX Writing (토스 보이스)

### 핵심 원칙
- **간결함:** 한 문장에 하나의 정보만. 조사와 접속사 최소화.
- **명확함:** 모호한 표현 금지. "잠시 후", "곧" 대신 구체적 시간.
- **친근함:** 경어체 사용, 하지만 과하게 격식 차리지 않음.
- **행동 중심:** 버튼 텍스트는 동사로 시작. "확인" 대신 "송금하기", "저장하기".

### 에러 메시지
```
❌ "오류가 발생했습니다"
✅ "인터넷 연결이 불안정해요. 다시 시도해 주세요."

❌ "유효하지 않은 입력입니다"
✅ "숫자만 입력할 수 있어요"
```

### 빈 상태 (Empty State)
- 왜 비어있는지 설명
- 다음 행동을 안내
- 일러스트와 함께 부드럽게 표현

## 2. Motion Design (모션 철학)

### Spring Animation
- **기본 설정:** `stiffness: 300, damping: 30` (자연스러운 바운스)
- **빠른 반응:** `stiffness: 400, damping: 40` (버튼, 토글)
- **부드러운 전환:** `stiffness: 200, damping: 25` (페이지 전환, 모달)

### Duration Guidelines
| 요소 | Duration | Easing |
|------|----------|--------|
| Micro (버튼, 토글) | 150-200ms | ease-out |
| Small (카드, 리스트) | 200-300ms | ease-in-out |
| Medium (모달, 시트) | 300-400ms | cubic-bezier(0.32, 0.72, 0, 1) |
| Large (페이지 전환) | 400-500ms | cubic-bezier(0.32, 0.72, 0, 1) |

### 의미 있는 모션
- **Enter:** 아래에서 위로, fade in (새로운 정보 등장)
- **Exit:** 위에서 아래로, fade out (정보 사라짐)
- **Expand:** 클릭 지점에서 확장 (컨텍스트 유지)
- **Collapse:** 원점으로 수축 (되돌아감 암시)

### 금지 사항
- 의미 없는 장식적 애니메이션
- 0.5초 이상의 대기 시간
- 사용자 행동을 방해하는 모션

## 3. Progressive Disclosure (점진적 노출)

### 원칙
1. **핵심 먼저:** 가장 중요한 정보/액션만 먼저 보여주기
2. **필요할 때:** 세부 정보는 사용자가 원할 때 노출
3. **단계별:** 복잡한 작업은 여러 단계로 분리

### 패턴 예시
```
1단계: 금액 입력
2단계: 받는 사람 선택
3단계: 최종 확인
(한 화면에 모든 것을 담지 않음)
```

### 정보 계층
- **Primary:** 즉시 보이는 핵심 정보 (잔액, 이름)
- **Secondary:** 스크롤/탭으로 접근 (상세 내역)
- **Tertiary:** 명시적 액션으로 접근 (설정, 약관)

## 4. Visual Hierarchy (시각적 위계)

### 여백 활용
- **섹션 간:** 32px 이상 (명확한 구분)
- **관련 요소 간:** 16px (그룹 내 요소)
- **밀접한 요소 간:** 8px (라벨-입력 필드)

### 포커스 포인트
- 한 화면에 하나의 주요 액션만
- CTA 버튼은 화면당 1개 권장
- 시선 흐름: 좌상단 → 우하단 (Z패턴)

### 그룹핑 (게슈탈트)
- 연관 정보는 카드로 묶기
- 구분선보다 여백으로 분리
- 색상/배경으로 영역 구분

## 5. Feedback & States

### 즉각적 피드백
- 터치 즉시 시각적 변화 (< 100ms)
- 로딩 중 스켈레톤 UI 표시
- 성공/실패 명확히 표현

### 상태별 스타일
```css
/* Default */
background: white;
border: 1px solid var(--grey-100);

/* Hover */
background: var(--grey-50);
transform: translateY(-1px);

/* Active/Pressed */
background: var(--grey-100);
transform: scale(0.98);

/* Disabled */
opacity: 0.4;
pointer-events: none;

/* Focus (접근성) */
outline: 2px solid var(--toss-blue);
outline-offset: 2px;
```

### 로딩 상태
- **즉시 (< 300ms):** 피드백 없음
- **짧은 대기 (300ms-1s):** 스피너 또는 shimmer
- **긴 대기 (> 1s):** 프로그레스 바 + 설명 텍스트

## 6. Mobile-First Patterns

### 터치 영역
- 최소 터치 타겟: 44x44px
- 권장 터치 타겟: 48x48px
- 인접 요소 간 간격: 8px 이상

### 엄지 영역 (Thumb Zone)
```
┌─────────────────────┐
│     Hard to reach   │  ← 덜 중요한 액션
│                     │
│   Natural reach     │  ← 탐색, 콘텐츠
│                     │
│   Easy to reach     │  ← 주요 액션, CTA
└─────────────────────┘
```

### Bottom Sheet 패턴
- 주요 액션 모음은 바텀 시트로
- 드래그로 닫기 가능
- 높이: 화면의 50% 이하 권장

### 제스처
- **스와이프 좌우:** 삭제, 아카이브
- **스와이프 아래:** 닫기, 새로고침
- **롱프레스:** 컨텍스트 메뉴

## 7. Accessibility (접근성)

### 색상 대비
- 본문 텍스트: 4.5:1 이상
- 대형 텍스트 (18px+): 3:1 이상
- UI 요소: 3:1 이상

### 키보드 네비게이션
- 모든 인터랙티브 요소 Tab 접근 가능
- 논리적 탭 순서 유지
- Skip link 제공 (메인 콘텐츠로 바로가기)

### 스크린 리더
- 의미 있는 `aria-label`
- 상태 변화 `aria-live` 알림
- 이미지에 대체 텍스트

## 8. 컴포넌트 패턴

### 버튼 계층
```
Primary   → 주요 액션 (화면당 1개)
Secondary → 보조 액션
Tertiary  → 텍스트 버튼 (취소, 더보기)
```

### 입력 필드
- 라벨 항상 표시 (placeholder만 사용 금지)
- 실시간 유효성 검사
- 에러 시 필드 아래에 빨간 텍스트

### 리스트
- 무한 스크롤보다 "더 보기" 버튼 선호
- 스켈레톤 로딩으로 레이아웃 시프트 방지
- 빈 상태 처리 필수

### 모달/다이얼로그
- 배경 딤 처리 (opacity 0.5)
- ESC 키로 닫기
- 외부 클릭으로 닫기 (설정에 따라)
- 포커스 트랩 (모달 내부에서만 탭 이동)

## 9. 성능 체감

### 체감 속도 향상
- Optimistic UI: 액션 즉시 반영, 실패 시 롤백
- 스켈레톤 UI: 레이아웃 미리 보여주기
- 프리페칭: 예상 경로 미리 로드

### 애니메이션 성능
- `transform`과 `opacity`만 애니메이션
- `will-change` 신중하게 사용
- 60fps 유지 (16.67ms/frame)

## 10. 체크리스트

### 디자인 검토
- [ ] 한 화면에 하나의 주요 목적이 있는가?
- [ ] 정보 계층이 명확한가?
- [ ] 여백이 충분한가?
- [ ] 색상 대비가 적절한가?

### 인터랙션 검토
- [ ] 모든 터치 요소가 44px 이상인가?
- [ ] 피드백이 즉각적인가?
- [ ] 에러 복구 경로가 있는가?
- [ ] 로딩 상태가 처리되어 있는가?

### UX Writing 검토
- [ ] 문장이 간결한가?
- [ ] 전문 용어 없이 이해 가능한가?
- [ ] 다음 행동이 명확한가?
- [ ] 에러 메시지가 친절한가?
